Accessibility Guidelines

Introduction

The W3C created guidelines to help developpers build accessible apps. It's a set of four topics, labeled as the POUR principles. They describe the main philosophy behind accessibility and are completed with best practices and typical cases.

This page sumarizes the essence of each principle, completed with one example each time. You can read the detailed version on the W3C website: Accessibility Principles.

The POUR principles

P for Perceivable

Create content that can be perceived by different senses. For example, use "alt" attributes in image tags to describe the scene. Images are part of the content, make it available for other senses than sight.

O for Operable

Ensure that the user interface can be operated through a variety of tools, not only the mouse. For example, make sure that the user can navigate with a keyboard or an eyetracker.

U for Understandable

Be clear and avoid confusion, make all the interactions predictable. For example, guide the user to correct the errors in a form.

R for Robust

Maximize compatibility with a wide range of assistive technologie tools. For example, use title tags from h1 to h6 to structure the page content, screen readers ofter option to sumarize the squeletton of the page.
